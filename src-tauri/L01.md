# L01/Lx01 Notes (EnCase Logical Evidence)

This document summarizes how CORE-FFX detects logical EVF containers.

## Signatures

From `src-tauri/src/ewf/types.rs`:

- L01 (LVF v1): `LVF\x09\x0d\x0a\xff\x00`
- Lx01 (LVF v2): `LVF2\x0d\x0a\x81\x00`

These share the EVF family structure but represent logical (file-based) evidence, not raw sector images.

## Detection

`ewf::is_l01_file` checks LVF/LVF2 signatures and section semantics to distinguish logical containers from E01/Ex01.

## Notes

- L01 is segmented (.L01, .L02, ...)
- Logical file metadata is stored in EVF sections rather than sectors
- Hashes may be stored per-file and/or container-level
